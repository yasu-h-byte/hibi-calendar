## 円未満の端数処理ポリシー（2026-06改定・全給与共通）

過去の労基署監査で「1円未満を四捨五入・切り捨てにすると支払額が過少」と指摘された経緯を踏まえ、
**支払額が下振れしない方向**で全項目を統一する（`lib/compute.ts` の `ceilYen` / `floorYen`）。

| 区分 | 処理 | 対象 |
|---|---|---|
| **支給項目** | **1円未満を切り上げ**（`ceilYen`） | 基本給・各種手当・残業代（残業単価も1円単位に切上）・追加所定・休業手当・法定休日手当・深夜手当 |
| **控除項目** | **1円未満を切り捨て**（`floorYen`） | 欠勤控除（切り上げると控除増＝過少支払いになるため逆方向） |
| **労働時間** | 0.1時間（6分）単位の四捨五入（従来どおり） | 残業時間など。時間の切り上げはしない |

- 浮動小数点ノイズ対策として、いったん銭（0.01円）単位に四捨五入してから円に丸める。
  例: `ceilYen(220000.0000001)=220000`、`ceilYen(2984.375)=2985`。
- 外注（協力業者）単価・歩掛比率は従業員給与ではないため対象外（従来どおり）。

## 日本人社員（日給月給制）

```
基本給   = ceil( 日額 × 実出勤日数 + 日額 × 0.6 × 補償日数 )
有給手当 = ceil( 日額 × 有給日数 )               ← 2026-06-12 追加（有給日も日給を支給）
残業単価 = ceil( 日額 ÷ 8h × 残業倍率(otMul) )   ← 1円未満を切り上げ
残業手当 = ceil( 残業単価 × 残業時間 )            ← 1円未満を切り上げ
支給額 = 基本給 + 有給手当 + 残業手当
```

- 残業単価の分母は **8時間**
- 残業倍率 (otMul) はデフォルト **1.25**（労基法37条準拠）
- **有給手当（2026-06-12 是正）**: 日給月給制は基本給が出勤日数連動のため、有給取得日が無給に
  なっていた（例: 白戸 5月有給4日 ¥85,200 未払い）。有給日も日給1日分を別途支給する。
- **端数処理（2026-06改定）**: → 「円未満の端数処理ポリシー」を参照（全給与共通）。
- 欠勤控除なし（出勤した日数分だけ支払う仕組み）
- **原価・netPay（2026-06-12 是正・監査C4）**: `totalCost = netPay = 支給額(salaryNetPay)` に統一。
  旧は丸め前 float の残業原価で数円ズレていた。
- 日本人で w=0.6 は半日勤務として workDays に 0.6 を加算（補償概念は外国人限定）

## 日本人社員（月給制）— 2026年6月〜追加

```
基本給 = 月給（固定、出勤日数に関わらず）
       ※月途中入退社のみ日割り（2026-06-12 追加・監査C5）: ceil(月給 × 在籍日数 ÷ 暦日数)
時給換算 = 月給 ÷ 145h（月平均所定労働時間・固定）
残業手当 = ceil( ceil(時給換算 × 残業倍率(otMul)) × 残業時間 )
支給額 = 基本給 + 残業手当
```

**月平均所定 145h の根拠（2026-07-09 変更・代表確定）**:
- 施行規則19条: 月ごとに所定労働時間が異なる場合、割増の基礎は「年平均の月所定労働時間」で除する
- 日比建設は年間休日が現場依存で事前確定できないため、新卒等の条件
  （完全週休2日・祝日出勤・GW/夏季/年末年始休暇、1日実労働7h）の想定レンジ
  「年間休日115〜125日 × 7h ⇒ 月平均140〜145.8h」の内側で **145h に固定**
- 分母を実際の所定より小さめに固定するのは単価が法定最低を上回る方向であり常に適法
- 旧方式（当月所定日数×8h）は単価が毎月変動し、契約実態（7h/日）とも不一致で法定割れの疑いがあった
- 例: 濱上（月給235,000円）: 235,000÷145 = 1,620.69円 → 残業単価 ceil(×1.25) = **2,026円/h**
- 定数: `lib/compute.ts` の `JP_SALARY_AVG_MONTHLY_HOURS`。変更時は賃金規程との整合も確認すること

- 月給制（完全月給）の判定: `salary` フィールドが 0 より大きい（例: 日比政仁、濱上祥太郎）
- **原価（2026-06 改定）**: 完全月給者の原価は**固定給そのもの**。
  `原価(totalCost) = 月給(固定) + 残業手当 = 支給額` とし、出勤日数に左右されない。
  - 旧: `cost = workDays × (salary/20)` の日額換算 → 出勤変動で固定給と乖離していた（廃止）。
  - 現場別配賦: 固定給を**出勤日数比**で各現場の原価へ配賦（単一現場なら全額）。出勤実績が
    無い月は配置現場の先頭へ計上。`lib/compute.ts` の `fullMonthlySiteDays` で実装。
  - 役員（事業責任者 政仁）の固定給も現場の労務費（原価・粗利）に含める運用。
- 切替方法: `/workers` 編集モーダルの「日給制／月給制」トグル
- 帳票表示: 月次集計Excel の日本人シートに「雇用形態」列（完全月給／日給月給）を表示。
  完全月給は「日額/月給」列に月給額を表示し、出勤日数に依らず基本給固定である旨を脚注に記載。

## ベトナム人スタッフ（3層構造・変形労働時間制）- 2026年5月〜

### 給与の4層構造（2026-06 改訂・2025-06再改訂）

**基本原則**: 全労働時間を「**基本給（1.0倍）+ 各種割増（追加分のみ）**」の足し算で構成する。

| 構成要素 | 計算方法 | 性質 |
|---------|---------|------|
| 基本給（固定） | 時給 × ベース日数(20日) × 7h | 1.0倍。毎月同額・減額なし（月給制） |
| 追加所定手当 | 時給 × MAX(0, 実出勤日数 − 20日) × 7h | 1.0倍。追加出勤日が発生した月のみ |
| **有給日給** | **時給 × (20日枠を超えた有給日数) × 7h** | **1.0倍。20日以上働いて有給も取った月のみ** |
| **所定外労働手当** | **時給 × 各日の所定時間超過分の合計** | **1.0倍。残業欄入力分（割増なし）** |
| **法定外残業手当** | **ceil(時給 × 0.25) × 法定外残業時間(0.1h丸め)** | **割増 0.25倍のみ（60h超は ceil(時給×0.5)）** |

> **有給日給（2026-06 社労士対応・月給制維持）**: 基本給は20日固定（月給制）のため、20日以上働いて
> 有給も取った人は、その有給が基本給枠に吸収されず未払いになっていた（例: ハウ +10,591円）。
> これを是正するため「20日枠を超えた有給日」を有給日給として別途支給する。
> `paidLeaveDays = MAX(0, (実出勤+有給)−20) − MAX(0, 実出勤−20)`。20日未満の人の有給は基本給20日枠に内包（別途支給なし）。
> あわせて、有給申請は「カレンダ上の出勤日のみ」に制限（非稼働日の有給による過払い防止）。

**重要（2025-06-XX 改訂）**: 法定外残業手当を「1.25倍 → 0.25倍（割増のみ）」に変更。

- **旧バグ**: 1.25倍（= 1.0倍base + 0.25倍割増）を支払っていたが、base 1.0倍は既に
  「基本給/追加所定/所定外労働手当」のいずれかで支払われていたため、二重支給になっていた
  - 例: 22日×7h勤務（週40h超で法定残業4h）→ 基本給+追加所定で154h × 1.0 + 法定外残業4h × 1.25
    = 159h相当 → 本来155h（154h + 4h×0.25）。**4h × 時給 / 月の過払い**
- **新ロジック**: 全労働を1.0倍ベースでカバーし、法定外残業に対しては割増分0.25倍のみ追加
  - 1.0h法定外残業 = 1.0倍（所定外労働手当） + 0.25倍（法定外残業手当） = 1.25倍 ✓

**端数・丸めの統一（2026-07-09・奥寺さん対応）**:
- **法定外残業時間を 0.1h 単位に丸めて金額計算**する（`Math.round(statutoryOT*10)/10`）。
  月次判定の法定上限（暦日×40÷7）が割り切れず端数（例 5.073h）が出るが、書類表示は 0.1h。
  金額を生値で計算すると「表示時間×単価」で手検算しても合わなかったため、表示と同じ値で計算する。
- **割増単価を先に1円未満切上げ**してから時間を掛ける（`ceil(時給×0.25) × 時間`）。
  例: 時給1581 × 0.25 = 395.25 → **396円/h**、396 × 5.1h → ceil = **¥2,020**。
  旧: `ceil(時給×0.25×時間)` は単価を先に丸める手計算と数円ズレていた。
- 月給制日本人の残業単価（`ceil(時給換算×otMul)×時間`）と丸め方針が揃った。

### 計算式

```
基本給（固定）  = 時給 × ベース日数(20日) × 7h         （1.0倍）
追加所定手当    = 時給 × MAX(0, 実出勤日数 − 20) × 7h   （1.0倍）
有給日給        = 時給 × 7h × 20日枠を超えた有給日数     （1.0倍。2026-06 追加）
所定外労働手当  = 時給 × Σ MAX(0, 各日actualHours − 7h) （1.0倍。残業欄入力分の合計）
法定外残業手当  = 時給 × 0.25 × 法定外残業時間（3層判定）+ 0.25×追加 (60h超)
法定休日手当    = 時給 × (1.35 × 8h以下 + 1.60 × 8h超)  （base + premium 合算）
深夜手当        = 時給 × 0.25 × 深夜労働時間            （割増のみ）
休業手当        = 時給 × 7h × 0.6 × 補償日数
欠勤控除        = 時給 × 7h × MAX(0, ベース日数 − 実出勤日数 − 有給日数 − 試験日数)
                  ※ 補償日は欠勤扱い（基本給から減額）、別途休業手当で60%補償
法定上限        = 暦日数 × 40 ÷ 7
支給額          = 基本給 + 追加所定 + 有給日給 + 所定外労働 + 法定外残業
                + 法定休日 + 深夜 + 休業 − 欠勤控除
```

> **週次の法定外残業判定（2026-06 社労士確認済み）**: 週の閾値は `MAX(40h, その週の事前所定合計)`。
> 事前カレンダーで週6日×7h=42h と定めた週は 42h 超のみが法定外残業（固定40hではない）。
>
> **salary（固定月給）設定の外国人**: 時給を `salary ÷ (ベース日数 × 7h)` で逆算し同じ式を適用。
> 基本給のみ月給値で固定（中途入退社は按分）。有給日給含む全手当が対象（2026-06-12 監査C1で加算漏れ是正）。

### 計算例（労基法準拠の検算）

**ケースA: 22日×7h勤務（週40h超で法定残業4h発生）**

時給1,500円のスタッフ:

| 項目 | 計算 | 金額 |
|---|---|---|
| 基本給 | 1500 × 20 × 7 | ¥210,000 |
| 追加所定 (2日分) | 1500 × 2 × 7 | ¥21,000 |
| 所定外労働 | 0（各日7h丁度、超過なし） | ¥0 |
| 法定外残業 (割増のみ) | 1500 × 0.25 × 4 | ¥1,500 |
| **合計** | | **¥232,500** |

検算: 実労働154h × 1.0倍 = ¥231,000 + 法定外残業4h × 0.25倍割増 = ¥1,500 → ¥232,500 ✓

**ケースB: 出勤18日+有給2日+残業18.5h（3層判定で法定外残業1h）**

| 項目 | 計算 | 金額 |
|---|---|---|
| 基本給 | 1500 × 20 × 7 | ¥210,000 |
| 追加所定 | 0（出勤18日 < ベース20日） | ¥0 |
| 所定外労働 | 1500 × 18.5 | ¥27,750 |
| 法定外残業 (割増のみ) | 1500 × 0.25 × 1.0 | ¥375 |
| **合計** | | **¥238,125** |

検算: 18日分実労働126h + 有給14h + 残業18.5h = 158.5h × 1.0倍 = ¥237,750 + 法定外残業1h × 0.25倍 = ¥375 → ¥238,125 ✓

### 所定外労働手当の意味

「基本給」は **月所定140h（=20日×7h）** のみをカバーする。
有給休暇日もこの枠に含まれる（取得しても基本給は減額されない）。

残業欄入力（出面の `o` 欄）に入力された時間は、**全て基本給とは別に支給** される:
- 3層判定（日/週/月）で法定外と認定された時間 → **法定外残業手当（1.25倍）**
- それ以外の時間 → **所定外労働手当（割増なし、通常賃金）**

> **例: 出勤18日 + 有給2日 + 残業合計18.5h、3層判定 statutoryOT=1.0h**
>
> - 基本給: 1,425円 × 20日 × 7h = ¥199,500
> - 所定外労働手当: 1,425円 × 17.5h = ¥24,938
> - 法定外残業手当: 1,425円 × 1.25 × 1.0h = ¥1,781
> - 合計支給額: ¥226,219

### 重要ルール
- ベース日数は管理者設定で変更可能（デフォルト20日）
- 実出勤日数に有給（P）は含めない
- 残業単価の分母は **7時間**
- 有給取得日は欠勤にカウントしない
- 欠勤控除はベース日数(20日)に対して判定
- **本システムの月次集計の残業手当は参考値**。最終的な残業判定はキャシュモさんが3段階で判定（下記参照）

### 残業の3段階判定（変形労働時間制）
法令上、変形労働時間制での残業は以下の順で段階的に判定する:
1. **1日単位**: 所定8h以下の日は8hを超えた分、所定8h超の日はその所定を超えた分
2. **1週単位**: **所定40h以下の週は40h超**、**所定40h超の週（週6日=42h等）は週所定超**（判定基準 = `max(40, 週所定)`。第1段階分は除く）
3. **1ヶ月単位**: 法定上限（暦日数×40÷7）を超えた分（第1・2段階分は除く）

> **2026-06 社労士確認・実装済み**: 事前にカレンダーで「週6日・42時間」と定めていれば、その42時間分の割増は不要。
> `lib/compute.ts:calculateVietnameseSalary` の週次判定を **週40h固定 → 週所定ベース（max(40, 週所定)）** に統一
> （社労士提出Excel側 `calculateOvertimeSummary` と同一）。月次集計の残業手当もこの3段階で算出する。

### HFU向け出面Excel（キャシュモ向け）
月次集計の「帳票出力」→「HFU向け出面一覧」で出力されるExcelには2シート含まれる:
- **Sheet 1「出面一覧」**: 従来形式（出勤1/残業h）— 入力確認・社内チェック用
- **Sheet 2「勤務時間一覧」**: 時間変換済み — キャシュモ向け給与計算用

  - 各日の実労働時間（所定7h + 残業h に自動変換）
  - 週番号（W1〜W5: 週単位の残業判定用）
  - 法定上限時間（暦日数×40÷7）
  - 計算方法の説明注記

## 旧ルール継続・固定月給（フン 104）— 2026-06-12 確定

本人希望で新ルール（変形労働時間制）へ移行しなかったスタッフ。`useOldRules=true` ＋ `salary` 設定で適用
（現在フン 104 のみ。2027年2月で現雇用契約終了）。

```
基本給     = 固定月給 396,105円（出勤日数・所定日数で変動しない）
             ※月途中入退社のみ日割り: ceil(396,105 × 在籍日数 ÷ 暦日数)
時給換算   = 日給(rate) 15,693 ÷ 6h40m(20/3h)   ← hourlyRate ではなく rate を使用（固定）
残業単価   = ceil(時給換算 × otMul) = 2,943円（固定。月給からの逆算ではない）
残業手当   = ceil(2,943 × 残業時間)
休業補償   = ceil(日給 × 0.6 × 補償日数)               ← 会社都合休(w=0.6)の60%支給分
補償日控除 = floor(日給 × 補償日数)                     ← 補償日の通常分（固定給は満額前提のため一旦控除）
欠勤控除   = floor(日給 15,693 × 欠勤日数)
欠勤日数   = MAX(0, 全社所定日数 − 実出勤 − 有給 − 試験 − 補償日)  ← 補償日は欠勤に含めない（所定日数はここにのみ影響）
有給       = 月給に内包（取得しても増減なし）
支給額     = 基本給 + 休業補償 + 残業手当 − 欠勤控除 − 補償日控除
```

- **会社都合休（補償日 w=0.6）の正味60%支給**: 固定月給は全所定日を満額前提とするため、補償日は
  「補償日控除（日給×1日・満額）」で一旦控除し、別途「休業補償（日給×0.6）」を支給する（労基法26条）。
  差引で補償日は**日給の40%控除＝60%支給**。**欠勤日数には含めない**（無給の欠勤だけが欠勤日数）。
  例: 5月所定23日・出勤20日・欠勤2日・補償日1日 → 欠勤日数=2、補償日控除=15,693、休業補償=9,416、
  支給額=396,105+9,416−31,386−15,693=**358,442**（補償日を単なる欠勤にした場合より日給の60%=9,416円多い）。

- 全社所定日数 = `main.workDays[ym]`（日曜以外 − 祝日。例: 2026年5月=23日、6月=26日）。
  現場カレンダー(siteWorkDays)は**使わない**（旧ルール継続者は全社所定が基準）
- 経緯: 当初「時給×所定日数×6h40m」で月変動 → 固定月給化 → 残業単価の月給逆算で単価が月変動
  → **日給ベースで全て固定**に是正（2026-06-12・3コミット）。詳細は historical-changes.md 参照

## 原価（労務費）の集計 — 2026-06 改定

月次集計・原価収益の「労務費（totalCost）」は、**実際の支給額（salaryNetPay）に統一**する。

| 雇用形態 | 労務費(totalCost) | 現場別配賦 |
|---|---|---|
| 日本人・日給月給 | **基本給 + 有給手当 + 残業 = 支給額**（2026-06-12 監査C4で完全一致に統一） | エントリ単位で加算（従来通り） |
| 日本人・完全月給 | **月給(固定) + 残業 = 支給額** | 支給額を出勤日数比で各現場へ配賦 |
| ベトナム人（新/旧ルール） | **3層構造の支給額（基本給+各手当−欠勤控除）** | 支給額を出勤日数比で各現場へ配賦 |

- 旧実装の問題: ベトナム人・完全月給の労務費が「日数 × 日額（+ 残業概算1.25倍）」で計算され、
  実際の支給額（3層構造／固定給）とズレていた。費用計算の精度に影響していた。
- 現実装: `lib/compute.ts` の `payrollSiteDays` で日本人日給月給以外（=ベトナム人+完全月給）を
  「支給額ベース」に統一。`wm.totalCost = salaryNetPay`、site.cost は支給額を出勤日数比で配賦。
- 月次集計テーブルの列名は「概算労務費」→「労務費」（実支給額ベースのため）。
- 例: グエン ミン トゥアン(102) 2026年5月は、旧概算 336,631 → 実支給額ベースに統一。

## 補償（0.6）の扱い
- 外国人の休業補償（w=0.6）: 会社都合休業日
  - **原価**: 計上する
  - **人工数**: 含めない
  - **基本給**: 欠勤扱いで1日分減額（時給×7h相当）
  - **休業手当**: 1日あたり 時給 × 7h × 0.6 = 60% を別途支給（労基法26条）
  - **正味受給**: 60%（労基法26条の最低ライン）
  - 2026-06-XX 修正: 旧実装は基本給で減額せず100% + 休業手当60% = 160% 支給していたため、労基法26条準拠の60%支給に訂正
- 日本人の0.6: 通常通りカウント（変形労働時間制で基本的に発生しない想定）

## 鳶合計・土工合計のルール
- 鳶合計 = とび + **鳶見習い (tobi_apprentice)** + 職長 + 役員 + 外注（鳶）
- 土工合計 = 土工 + 外注（土工）
- 出向者は鳶換算人工（tobiEquiv）から除外するが、出面合計には含む
- 鳶見習いは `lib/jobs.ts` の `TOBI_GROUP_JOBS` 配列で管理 → 集計ロジックは `isTobiGroup(job)` に集約

## 退職月のスタッフの扱い
- `lib/workers.ts:isStillActiveForMonth(retired, ym)` で判定
- 退職日 >= 表示月の月初 なら **その月は集計対象**
  - 例: 退職日 = 2026-06-30 → 2026年6月の集計には含まれる、7月以降は除外
- 出面入力・カレンダー署名・月次集計・Excel 出力すべてで同じヘルパーを使用

## 月次給与計算の監査機能

### 計算根拠表示モーダル
`/monthly` ページのスタッフ名をクリックすると「🔍 給与計算の根拠」モーダルが開く:

1. **月情報**: 暦日数 / 法定上限h / 所定日数
2. **雇用区分・適用ルール**: 区分（月給/日給/時給）/ 旧 or 新ルール / 単価情報
3. **出勤実績の集計**: 出勤・補償・有給・欠勤・残業・深夜・法定休日
4. **計算式と結果**: 基本給 / 追加所定 / 残業 / 法定休日 / 深夜 / 欠勤控除 / 支給額
5. **監査チェック**: ✓/❌ で表示
   - 所定労働時間が法定上限以内
   - 出勤実績の合計が所定日数以内
   - 支給額の内訳合計が一致
   - 残業倍率が法定下限以上
