## A. 最重要（要決定・給与額に影響）

### A-1. 週6日勤務（笹塚=週42h）の残業判定 ★最重要
- **現状**: 週次の時間外を「週40時間超」で集計している。
- **論点**: 変形労働では、所定が40hを超える週は「**週所定（42h）を超えた分**」が時間外のはず。週42hで判定してよいか。
- **影響範囲**: 週6日現場（笹塚）のスタッフのみ。5日週の IHI 等は影響なし。
- **2026年5月の実額**（`npm run diagnose:hours 202605` で算出）:

| スタッフ | 現状(週40h固定) | 週所定42h | 差 | 過払い割増(0.25倍) |
|---|---|---|---|---|
| ファン スアン ハウ(203) | 4h | 0h | 4h | 約¥1,513 |
| ルオン バオ クオック(204) | 6h | 0h | 6h | 約¥2,138 |
| ホー チョン ゴック(205) | 6h | 0h | 6h | 約¥2,138 |
| **合計** | | | **16h** | **約¥5,789** |

- 方向は**会社の過払い是正**（払いすぎを正す方向）。

### A-2. 変形労働時間制の協定整備（A-1の前提）
- [ ] (a) 変形労働時間制を**就業規則 または 労使協定**で定めているか（常時10人以上は就業規則必須）
- [ ] (b) 笹塚等の週6日週の所定（週42h）を、**期間開始前にシフトで特定・周知**しているか
  - 当社運用: 前月25日までに就業カレンダー提出 → 事業責任者承認 → 本人署名
- [ ] (a)(b)が満たされる場合、残業を**「週42h超」で判定**してよいか

### A-3. 2026年5月分の再計算・是正
- [ ] A-1がYESの場合、5月分（6/25支給）を**週所定ベースで再計算して是正**してよいか

## C. 運用・端数処理

### C-1. 休憩未取得分の労働時間算入（労基法34条との関係）
- 富岡等で、作業内容により10時・15時の休憩を取れない日がある。その分は**休憩を取らず労働時間に算入**し、出面の休憩チェックボックスで「取得／未取得」を記録している。
- [ ] この運用（休憩未取得分＝労働時間）の整理で問題ないか。34条で留意点があれば。

### C-2. 円未満の端数処理（過少支払い防止）
- 過去の労基署監査で「1円未満の四捨五入・切り捨ては支払額が過少」と指摘された経緯を踏まえ、以下に統一：
  - **支給項目**（基本給・各種手当・残業代）→ **1円未満を切り上げ**
  - **控除項目**（欠勤控除）→ **1円未満を切り捨て**
  - 労働時間 → 0.1時間（6分）単位の四捨五入
- [ ] この端数処理（従業員有利方向）で問題ないか。

### C-3. 出面の二重入力（データ整理・給与額への影響なし）
- 一部エントリで「始業・終業(st/et)」と「残業欄(o)」が両方入力されている例がある。給与計算は st/et を優先し o を無視するため**支給額には影響しない**。
- [ ] 上記の理解（st/et 優先で o 無視）で問題ないか。

## E. 当社で既に対応済み（参考）

- 実労働明細Excel: 休憩控除を給与計算と統一（時間の二重基準を解消）
- 勤務予定シフトExcel: **週所定列・週合計**を追加（変形労働の週所定を明示）
- 月次集計・PDF: 残業表示を計算値（所定外+法定外）に統一
- 端数処理: 支給=切上／控除=切捨 に統一（C-2）
- 診断スクリプト `npm run diagnose:hours`（週40h論点の影響者・金額を抽出）
