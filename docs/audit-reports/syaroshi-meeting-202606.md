## 2. 確認・承認をお願いしたい論点

### 論点B（最重要）: 週6日勤務の週所定と残業判定

**現状**: 給与計算が週次残業を「週40h超」で一律判定している。
**論点**: 変形労働では、所定が40hを超える週は「週所定超」が時間外のはず。笹塚（週6日=週42h）なら、残業は「週42h超」で判定すべきではないか。

確認事項:
- [ ] (a) 変形労働時間制を**就業規則 または 労使協定**で定めているか（常時10人以上は就業規則必須）
- [ ] (b) 笹塚等の週6日週の所定（週42h）を、**期間開始前にシフトで特定・周知**しているか（当社は前月25日までに就業カレンダー提出→事業責任者承認→スタッフ署名）
- [ ] (a)(b)が満たされる場合、残業を**「週42h超」で判定**してよいか
  - → YES なら給与計算を週所定ベースに修正（社労士提出Excel側は既に正しい実装あり）

**影響額（2026年5月・実数）**:

| ファン スアン ハウ(203) | 4h | 0h | 4h | 約¥1,513 |

- いずれも**HFU**。日比建設は週6日のベトナム人がいないため影響なし。
- 方向は**会社の過払い是正**（払いすぎを正す方向）。
- 診断: `npm run diagnose:hours 202605`（read-only）

### 論点D: 2026年5月分の再計算・是正

- [ ] 論点BがYESの場合、5月分（6/25支給）を週所定ベースで**再計算して是正**してよいか

### 論点A: 休憩未取得分の労働時間算入（念のため確認）

- 富岡等で10時・15時の休憩を取れない日は、その分（各30分）を労働時間に算入し、休憩チェックボックスで「取得/未取得」を記録している。
- [ ] この運用（休憩未取得分=労働時間）の整理で問題ないか。34条（休憩）との関係で留意点があれば。

### 論点C: 出面入力の二重入力（データ品質。支給額への影響なし）

- 一部エントリで「始業・終業（st/et）」と「残業欄（o）」が両方入力されている（例: グエン タイン フウ 101 は毎日）。
- 給与計算は st/et を優先し o を無視するため**支給額には影響しない**が、入力時に併用を防ぐバリデーションを追加予定。
- [ ] 上記の理解（st/et 優先で o 無視）で問題ないか。

### 論点E: 法定休日割増の検証

- 日曜（法定休日）の出勤に 1.35倍を適用。深夜 0.25倍、休業 60%。
- [ ] 割増率・適用範囲に問題ないか。
