# 出面入力

## 入力値
| 値 | 意味 | 色 |
|----|------|-----|
| 1 | 出勤（1日） | 緑 |
| 0.5 | 半日 | 黄 |
| P | 有給休暇 | 紫 |
| E | 試験（exam） | 青 |
| HK / ✈️ | 帰国期間（home_leave） | 水色 |
| R | 休み（rest） | グレー |
| H | 祝日扱い | グレー |
| 補（0.6） | 休業補償 | 紫 |
| 下段の数字 | 残業時間（0.5h単位） | オレンジ |
| 空欄 | 休み / 未入力 | — |

### AttendanceStatus 型と優先順位

`AttendanceStatus` 型: `'work' | 'overtime' | 'p' | 'compensation' | 'rest' | 'holiday' | 'home_leave' | 'exam' | 'empty'`

- `'home_leave'`（帰国期間）と `'exam'`（試験）を追加
- **UI 表示の優先順位（`getEntryStatus`）**: P > E > R > H > HK > 残業 > 出勤 > 未入力
  - スタッフ向け表示で「個別の休みの理由」を優先するため、HK より R/H が先
- **月次集計の優先順位（`lib/compute.ts`）**: HK が R/H より先に continue
  - 人工計算上、帰国期間中は労務集計の対象外として早期スキップする方が整合的

UI と集計でロジックが分離されているため、両者を改修するときは必ず両方の優先順位を確認すること。

## PC画面（管理者向け）
- 現場ごと・月ごとのグリッド入力
- 自動保存（入力するだけでOK）
- 配置編集ボタンでスタッフの現場割り当て変更
- 終了現場は「終了現場」チェックボックスで表示
- 年月セレクタ: 26ヶ月分（2024年4月まで遡れる）
- 日付ヘッダーはsticky固定（下にスクロールしても表示）
- 名前列・所属列はsticky固定（右にスクロールしても表示）

## スマホ画面（スタッフ向け）
- トークンURLでアクセス（ログイン不要）
- 「しゅっきん」ボタンで出勤登録（残業時間も設定可能）
- 「やすみ」ボタンで休み登録
- 「有給申請」ボタンで有給休暇を申請（残日数チェックあり）
- 過去5日分の入力履歴を確認・修正可能
- 日本語とベトナム語の二言語表示

### 出勤登録時の休憩チェックボックス
- 「午前休憩を取得した」「午後休憩を取得した」の2つのみ表示
- **昼休憩のチェックボックスは削除**（現場の昼休憩は基本的に必須のため、内部的に取得扱いで固定）
- チェック状態は実労働時間の計算に反映（後述の現場別勤務時間設定と連動）

### 道具代・有給残カード
- データがあれば即座に表示する（運用フェーズによるガードはなし）
- 道具代の控除期間表記は西暦付き `'2026/2/7 〜 2027/2/6'` 形式に統一

## 5月以降の運用フロー
1. 実習生・特定技能がスマホから毎日入力
2. 政仁さんがPC画面で確認・修正・承認
3. 奥寺さんが月次集計で最終確認 → 月締め

## フッター合計のルール
- 鳶合計 = とび + 職長 + 役員 + 外注（鳶）
- 土工合計 = 土工 + 外注（土工）
- 総合計 = 鳶合計 + 土工合計
- 外国人の補償（w=0.6）は人工数に含めない（compute()と統一）
- 浮動小数点は丸め処理（Math.round(n*10)/10）

## 出面の承認フロー（3段階）
1. **スタッフ**がスマホから出勤/休みを入力
2. **職長**がスマホ職長画面で確認・修正 →「この日を確認する」→ スタッフ入力ロック
3. **事業責任者（政仁さん）**がPC画面の「承認」行をクリックして最終承認

### 承認の影響
- 職長が確認 → スタッフの入力がロック（職長・管理者は編集可能）
- PC画面の承認行 → admin/approverロールがクリック可能（承認/承認解除）
- 承認データは `attendanceApprovals/{siteId}_{ym}_{day}` に保存

## 出面ソース追跡
- s='staff': スタッフがスマホから入力
- s='foreman': 職長が入力
- s='admin': 管理者がPC画面から入力

## 現場別勤務時間設定

`Site` 型に `workSchedule` フィールドを追加し、現場ごとに始業・終業・休憩を上書き可能にした。

```typescript
workSchedule?: {
  startTime: string                                            // 例: '07:30'
  endTime: string                                              // 例: '17:30'
  morningBreak:   { enabled: boolean; minutes: number; mandatory: boolean }
  lunchBreak:     { enabled: boolean; minutes: number; mandatory: boolean }
  afternoonBreak: { enabled: boolean; minutes: number; mandatory: boolean }
}
```

### デフォルト（未設定の現場）
- 始業 8:00 / 終業 17:00
- 午前休憩 30分 / 昼休憩 60分 / 午後休憩 30分
- 1日の所定実労働時間 = 7h

### IHI現場（設定例）
- 始業 7:30 / 終業 17:30（10時間拘束）
- 休憩構成は会社デフォルトを踏襲
- スマホ出勤画面でこの現場を選択すると、表示される勤務時間が 7:30〜17:30 に切り替わる

### 連動する計算ロジック
- `types/index.ts` の `calcActualHours` / `calcOvertimeHours` は `workSchedule` を引数で受け取り、現場別の所定時間で実労働時間と残業時間を算出
- `lib/compute.ts` の月次集計も同じ workSchedule を参照して所定・残業を判定
- スマホ出勤登録の休憩チェックボックスは「enabled かつ mandatory ではない」休憩のみ表示
