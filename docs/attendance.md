# 出面入力

## 入力値
| 値 | 意味 | 色 |
|----|------|-----|
| 1 | 出勤（1日） | 緑 |
| 0.5 | 半日 | 黄 |
| P | 有給休暇 | 紫 |
| 補（0.6） | 休業補償 | 紫 |
| 下段の数字 | 残業時間（0.5h単位） | オレンジ |
| 空欄 | 休み / 未入力 | — |

## PC画面（管理者向け）
- 現場ごと・月ごとのグリッド入力
- 自動保存（入力するだけでOK）
- 配置編集ボタンでスタッフの現場割り当て変更
- 終了現場は「終了現場」チェックボックスで表示
- 年月セレクタ: 26ヶ月分（2024年4月まで遡れる）
- 日付ヘッダーはsticky固定（下にスクロールしても表示）
- 名前列・所属列はsticky固定（右にスクロールしても表示）

## スマホ画面（スタッフ向け）
- トークンURLでアクセス（ログイン不要）
- 「しゅっきん」ボタンで出勤登録（残業時間も設定可能）
- 「やすみ」ボタンで休み登録
- 「有給申請」ボタンで有給休暇を申請（残日数チェックあり）
- 過去5日分の入力履歴を確認・修正可能
- 日本語とベトナム語の二言語表示

## 5月以降の運用フロー
1. 実習生・特定技能がスマホから毎日入力
2. 政仁さんがPC画面で確認・修正・承認
3. 奥寺さんが月次集計で最終確認 → 月締め

## フッター合計のルール
- 鳶合計 = とび + 職長 + 役員 + 外注（鳶）
- 土工合計 = 土工 + 外注（土工）
- 総合計 = 鳶合計 + 土工合計
- 外国人の補償（w=0.6）は人工数に含めない（compute()と統一）
- 浮動小数点は丸め処理（Math.round(n*10)/10）

## 出面ソース追跡
- s='staff': スタッフがスマホから入力
- s='foreman': 職長が入力
- s='admin': 管理者がPC画面から入力
