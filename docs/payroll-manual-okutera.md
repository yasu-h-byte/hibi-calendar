# 給与計算マニュアル（奥寺さん用）

> **対象**: 日比建設株式会社の月次給与計算担当（奥寺さん）
> **担当範囲**: **日比建設のみ**（HFU の給与計算は靖仁さんが外注に委託、対象外）
> **使用システム**: HIBI CONSTRUCTION 就業カレンダーシステム
> **作成日**: 2026年6月
> **最終更新**: 2026年6月

---

## 0. はじめに

このマニュアルは、月次給与計算で **奥寺さんが行う操作と確認項目** をまとめたものです。

### 担当範囲の整理

| 対象 | 担当者 | 給与計算方法 |
|---|---|---|
| **日比建設の従業員** | **奥寺さん** ← このマニュアルの対象 | 本システムで集計 → 振込手続き |
| HFUの従業員 | 靖仁さん | 別途、外注業者に委託 |
| 外注業者（協力会社） | 政仁さん経由 | 業者向け請求書ベース |

→ **奥寺さんは「日比建設」タブのみ見ればOK** です。HFU タブ・外注タブは見なくて大丈夫です。

### 給与計算の責任分担

```
[出面入力] → [カレンダー承認] → [月次集計確認] → [Excel出力] → [振込手続き]
スタッフ・職長   職長提出→政仁承認     ★ 奥寺さん ★     ★ 奥寺さん ★    ★ 奥寺さん ★
   (前月末まで)     (前月末まで)        (毎月10日まで)    (10〜20日)      (25日まで)
```

奥寺さんは **「月次集計確認」以降** を担当します。
出面の入力・修正に違和感があれば 靖仁さんに連絡してください。

| 列名 | 意味 | 注意点 |
|---|---|---|

| **所属** | 「日比」バッジ | HFUは別タブで表示 |

| **出勤日数** | 月間出勤日数（補償日0.6含む） | 補償日があれば「うち補 0.6」と注釈 |

| **追加所定** | 21日目以降の出勤日 × 7h × 時給 | 外国人新ルールのみ |
| **所定外労働** | 1日の所定(7h)を超えた残業 × 時給 | 外国人新ルールのみ、1.0倍 |

## 3. ⚠ 自動検算バナーが出ているとき

画面上部に **赤バナー** または **黄バナー** が出ている場合は **要確認**:

```
⚠ 給与計算に異常があります（1名 / 検出1件）
労基法・実労働時間ベースの自動検算で critical 1件 を検出しました。
該当スタッフの行クリックで「計算根拠」モーダルを開いて詳細を確認してください。

❌ ハウ: 法定外残業手当が上限（0.5倍）を超える（二重支給の可能性）
  (想定 ¥1,513 / 実額 ¥7,500)
```

### 対処方法

1. 該当スタッフの行をクリック → 計算根拠モーダルを開く
2. 「⑤ 監査チェック」セクションで ❌ になっている項目を確認
3. 出面入力に不自然な点がないか確認（残業時間の入力ミスなど）
4. 必要に応じて靖仁さんに連絡

**赤バナー (critical)** = 計算ロジック上の異常。必ず確認。
**黄バナー (warning)** = 通常範囲を逸脱した値。チェック推奨。

### バナーが出ていない場合 → 全員 OK

```
（バナーが画面に表示されない = 全スタッフの計算は法令準拠で問題なし）
```

## 6. 6パターンの給与計算式（参考）

スタッフごとに、雇用区分により計算式が変わります。

### パターン1: 日本人 日給制（多くの日比建設従業員）

```
基本給 = 日額 × 出勤日数 + 日額 × 0.6 × 補償日数
残業手当 = (日額 ÷ 8h) × 1.25 × 残業時間
支給額 = 基本給 + 残業手当
```

### パターン2: 日本人 月給制

```
基本給 = 月給（固定値）
残業時給 = 月給 ÷ (月所定日数 × 8h)
残業手当 = 残業時給 × 1.25 × 残業時間
支給額 = 基本給 + 残業手当
```

### パターン3: 外国人 時給制 新ルール（多数派ベトナム人）

```
基本給(固定)    = 時給 × 20日 × 7h                ← 全社固定
追加所定手当    = 時給 × MAX(0, 出勤-20) × 7h
所定外労働手当  = 時給 × 残業欄入力時間 × 1.0倍   ← base
法定外残業手当  = 時給 × 法定外残業時間 × 0.25倍 ← premium
法定休日手当    = 時給 × 日曜実労働 × 1.35倍
深夜手当        = 時給 × 0.25 × 深夜労働h
休業手当        = 時給 × 7h × 0.6 × 補償日
欠勤控除        = 時給 × 7h × 欠勤日数

支給額 = 上記合計 − 欠勤控除
```

> 💡 「追加所定」と「所定外労働」の違い:
> - 追加所定 = **日数**ベース（21日目以降の丸1日）
> - 所定外労働 = **時間**ベース（1日の所定 7h を超えた残業）
> - どちらも 1.0倍だが集計の粒度が違うだけ

### パターン4: 外国人 月給制 新ルール

```
パターン3と同じ式。基本給だけ「月給固定」で時給は salary ÷ 140h で逆算。
```

### パターン5: 外国人 時給制 旧ルール継続（フンさん 104）

```
基本給         = 時給 × 月所定日数 × 6h40m  ※5月なら 23 × 6.667 = 153.33h
休業補償       = 時給 × 6h40m × 0.6 × 補償日
残業手当       = 時給 × 1.25 × 全残業時間  ※3層判定なし、1.25倍で一括
欠勤控除       = 時給 × 6h40m × 欠勤日数

支給額 = 基本給 + 休業補償 + 残業手当 − 欠勤控除
```

### パターン6: 外国人 月給制 旧ルール継続

```
パターン5と同じ式。基本給だけ月給固定。
```

## 7. よくある問題と対処（FAQ）

### Q1. スタッフが画面に出てこない

**A.** 以下のいずれか:
- 退職日が月初より過去 → 人員マスタで `retired` フィールドを確認
- 当月一切出勤も有給もなし → 表示対象外（給与計算なし）

### Q2. 基本給が ¥0 になっている

**A.** スタッフの人員マスタで `hourlyRate` と `salary` が両方未設定の可能性。
靖仁さんに連絡して人員マスタを確認。

### Q3. 残業を入れていないのに「残業手当」が出ている

**A.** 週40時間超による **自動判定** です。
例：月〜土の6日勤務で、各日 7h勤務 = 週42h → 週40h超分の 2h が法定外残業に。

これは労基法準拠の自動判定です。正常動作。

### Q4. 補償日 (w=0.6) があるスタッフの支給額が想定と違う

**A.** 補償日は 60% 支給（労基法26条）。
- 基本給からは 1日分 減額（欠勤控除）
- 別途、休業手当 = 時給 × 7h × 0.6 を加算
- **正味: 通常日の 60% の支給**

### Q5. 旧ルール継続スタッフ（フンさん）の計算が他の人と違う

**A.** 正しい動作です。
- フンさんは本人希望で旧ルール継続を選択
- 月所定時間ベース（時給 × 月所定日数 × 6h40m）で基本給算出
- 残業は全時間 × 1.25 倍
- 退職まで継続

### Q6. 中途入社・退社月の計算は？

**A.** 在籍日数を暦日数で按分。
- 例：6/15入社（在籍 16/30 = 53%）→ baseDays = round(20 × 53%) = 11日
- 基本給も同比率で減額（月給制の場合）

### Q7. 計算結果が「想定と違う」感じがする

**A.** 以下の順で確認:
1. 該当スタッフの行をクリック → 計算根拠モーダル
2. 「① 月情報」の所定日数が正しいか
3. 「② 雇用区分」が正しいか
4. 「③ 出勤実績」が出面と一致しているか
5. 「⑤ 監査チェック」で 5/5 すべて ✓ か

それでも不明なら靖仁さんに連絡。

### Q8. Excelと画面の数字が合わない

**A.** ブラウザのキャッシュの可能性:
1. ブラウザをリロード（Cmd+R / Ctrl+R）
2. 再度 Excel をダウンロード

### Q9. 振込明細を社内会計システムに入れる際の注意

**A.** Excel から **支給額合計** の列を見て転記してください。
- 列名: `支給額合計`
- これは「基本給 + 各手当 − 欠勤控除」が全て計算済みの最終金額
- そのまま振込金額として使えます

## 11. 補足: 印刷・PDF出力について

`/monthly` 画面に「📄 社労士確認用PDF」というボタンがあります。
これは **社労士チェック用のPDF出力機能** で、ベトナム人スタッフの計算根拠を一覧で見られます。

- 日比建設のベトナム人がいる場合、社労士確認時に使用
- 通常は政仁さん・靖仁さんが操作するので、奥寺さんが触る必要は基本的にありません
- もし政仁さんから「社労士に送るPDFを出して」と依頼があれば、「日比建設」ボタン → Cmd+P → PDF として保存
