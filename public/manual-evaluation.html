📢 提出が成功したか分かるようになりました（2026-05-09 改善）

「提出する」を押すと、以下 3つの形 で「保存できた」が確認できます：

  1. 確認ダイアログが画面中央に出る — クリックで閉じるまで消えません
    ✅ 〇〇さんの評価を提出しました
  2. 緑のトーストバナーが画面最上部に表示（8秒で自動フェード）
  3. 画面が自動的にスクロールトップし、「✓ 〇〇さんの評価を提出済みです（保存完了時刻）」の緑バナーが表示される

⚠️ これらが出なければ、保存されていません。もう一度「提出する」を押してください。

提出後にできること：
  • 自分の評価を修正したい場合は「修正する」ボタンで再編集可能
  • 他の評価者の結果を閲覧可能（提出前は見えません）
  • 他の対象スタッフがいる場合は、上のドロップダウンから次の人を選んで続けて評価できます
複数のスタッフを評価する場合の注意：
1人提出した後、必ずドロップダウンから次のスタッフを選んで評価を続けてください。同じ画面に居続けても次のスタッフには切り替わりません。
進捗を確認したい場合は「一覧」タブで提出状況バッジを見ると一目です。

6. 提出状況の確認方法 NEW

「誰が提出済みで、誰がまだか」を確認する方法は 3つあります。

① 一覧タブ — 全スタッフを横断的に確認

「提出状況」列の評価者バッジを見れば、各セッションの状況が一目で分かります。

  • ✓田中 緑＝提出済み（マウスオーバーで提出日時）
  • ○山田 グレー破線＝未提出
  • ○大川 橙＝7日以上未提出（停滞警告）
  • 👤✓政仁 青枠＝自分

② 進捗監視タブ（管理者専用） — 進行中セッションだけを2カラムで一望

収集中・最終確認待ちのセッションだけを横並びカード表示。停滞中のセッションは橙ボーダーで強調されます。

③ 詳細モーダル — 1セッションを掘り下げ

一覧の「詳細」ボタン、または進捗監視のカード「詳細」ボタンから開けます。表示内容：

  • 提出状況バッジ（評価者ごと）
  • 出勤実績（過去1年）
  • 評価者ウェイト一覧（共働率・ウェイトバー）
  • 評価者比較表（読み取り専用）
  • 各評価者のコメント
大川さんが「ゴック・サンの評価をしたつもりだったが保存されていなかった」事案を受けて、これらの可視化機能を追加しました。 今後は提出漏れがあれば一覧バッジで即座に発見できます。

7. 評価者ウェイト（重み付き多数決）NEW

2026-05-09 導入：「直近で実際に一緒に現場にいた職長の意見」が多数決プリフィルで強く反映される仕組み

なぜ必要？

これまでは、対象スタッフを毎日見ている主担当の職長と、たまに顔を出した程度の職長の意見が 同じ重み（1票） で扱われていました。実態に即した評価のため、共働実績に応じて重みを変えるようにしました。

ウェイトの計算式

事業責任者（靖仁さん・政仁さん）: weight = 1.0（固定） ← ★ マーク付き

その他職長: 
recent = min(直近90日の共働日数, 60)
year   = min(過去365日の共働日数, 200)
weight = 0.3 + 0.4 × (recent / 60) + 0.3 × (year / 200)
範囲: 0.3 〜 1.0

シナリオ別の例

シナリオ直近90日過去365日weight
主担当職長（毎日同じ現場）60+ 日200+ 日1.0
数ヶ月一緒だったが直近離れた0 日100 日0.45
直近たまに一緒20 日30 日0.48
一度も同じ現場なし0 日0 日0.30
政仁さん・靖仁さん――1.0 ★

共働日数の判定方法

対象スタッフが「その評価者が月の職長として責任を持っていた現場」で実出勤した日をカウントします。

  • 月単位の職長変更（mforeman）も忠実に反映
  • 有給・休み・帰国中などの非作業日は除外

使われ方：承認画面の多数決プリフィル

承認時に最終ABCを決める際、これまでは 1人=1票 の多数決で初期値が決まっていました。今後は 1人=ウェイト分の票 で初期値が計算されます。

例：3名の評価者がそれぞれ A / A / B と評価し、ウェイトが 0.95 / 0.45 / 1.0 の場合
A票合計 = 0.95 + 0.45 = 1.40 / B票合計 = 1.0 → A が初期値になる
あくまで「初期値（プリフィル）」であり、最終決定は政仁さん（または管理者）が承認画面で自由に変更できます。運用の最終責任は変わりません。

ウェイトの確認方法

承認画面の比較表ヘッダーと、詳細モーダルの「評価者ウェイト」セクションで確認できます：

  • w=0.95 / 直近63% / 年71% のような表示
  • ★マークは事業責任者として 1.0 固定の意
  • マウスオーバーで実日数も表示

ウェイトの再計算（管理者）

  • 一覧タブ右上の「🔄 ウェイト一括再計算」ボタン：進行中セッション全部のウェイトを最新の出面データから再算出
  • 詳細モーダル内の 「🔄 再計算」ボタン：個別セッションだけ再算出
承認済みは再計算対象外です（既に最終決定済みのため、変更しない設計）。

8. 評価の承認（政仁さん・管理者）

全評価者の提出が完了すると、「承認」タブに対象スタッフが表示されます。

1
「承認」タブを開く（または一覧から「承認へ」ボタンで直接）
2
「確認・承認」ボタンを押す → 重み付き多数決でプリフィルされた最終ABCが表示される
3
比較表が表示される：各評価者の回答が横並びで確認できます
● 緑 = 全員一致、● 黄 = 意見が分かれている項目
ヘッダーの w=0.85 は評価者ウェイト
4
必要に応じて最終ABCを変更する（A/B/Cボタンをクリック）
5
最終コメントを記入
6
スコア・ランク・昇給額のプレビューを確認し、「承認する」ボタンを押す
承認すると：
  • 確認ダイアログ ✅ 〇〇さんの評価を承認しました（ランク: A / 推奨昇給: +170円/h） が表示される
  • 最終スコア・ランク・昇給額が確定
  • 履歴タブから後日いつでも詳細を閲覧可能
  • 昇給額は人員マスタの時給に手動で反映してください

9. 進捗監視と履歴閲覧 NEW

  • 名前 / 評価日 / 勤続 / 合計スコア / ランク / 推奨昇給 / 承認日
  • 「詳細」ボタンで承認時の評価者比較表・コメント・最終結果を再閲覧可能
  • 過去の評価傾向の比較・1on1での振り返りに活用できます

11. 評価基準（ABC判定の目安）

評価項目の構成：全10項目（日本語3 + 勤務態度4 + 職業能力3）
勤務態度が4項目かつ重み×1.5のため、スコアの約半分（48%）を占めます。
真面目に取り組む姿勢が最も重視される設計です。

🗣 日本語能力（3項目 × 重み ×1.0 = 24%）

項目ABC
指示理解 一度で正確に理解し、確認の質問もできる 概ね理解するが、たまに聞き返しが必要 繰り返し説明が必要。複雑な指示は通じにくい
報告・連絡 自分から日本語で報告・連絡ができる。問題発生時も的確に伝えられる 聞かれれば答えられるが、自発的な報告は少ない 通訳が必要な場面が多い。報告が遅れることがある
安全用語 危険予知・安全指示を理解し、他のスタッフにも伝達できる 基本的な安全用語（危ない・止まれ等）は理解 安全に関する指示の理解が不十分

💼 勤務態度（4項目 × 重み ×1.5 = 48% — 最重要）

項目ABC
時間厳守 無遅刻・無欠勤。始業前に準備を済ませている 年1〜2回の遅刻程度。基本的には時間を守る 遅刻・無断欠勤が目立つ
安全意識 安全帯の着用を自ら徹底。周囲の安全にも気を配る 注意すれば守る。基本的なルールは理解 繰り返し注意が必要。不安全行動がある
協調性 他のスタッフと積極的に協力し、後輩の面倒も見る 与えられた仕事を問題なくこなす 単独行動が多い。チームワークに課題
指示遵守 職長や先輩社員の指示を正確に守り、自己判断で勝手に変えない。不明点は必ず確認する 基本的に指示は守るが、たまに自己判断で省略・変更することがある 指示を守らないことが目立つ。自分のやり方で勝手に進めてしまうことがある

🔨 職業能力（3項目 × 重み ×1.0 = 20%）

項目ABC
技能レベル 一人で任せられる作業が多い 指示を受ければ一通りの作業ができる 常に監督が必要。ミスが多い
作業速度・品質 日本人と同等以上。手直し不要 概ね問題なし。時々確認が必要 改善が必要。手直しが頻繁
段取り・準備 次の工程を予測して自ら準備できる 言われれば段取りできる 段取りの意識が低い。指示待ち

🏠 生活態度（3項目 × 重み ×1.0 = 20%）

生活態度は管理者（靖仁さん）のみが入力します。
靖仁さんがエムテック協同組合の担当者（チイさん）と一緒に確認し、代理入力します。
職長・政仁さんの評価入力画面には表示されません。
項目ABC
近隣配慮 近隣からの苦情ゼロ。騒音・駐車等に自ら配慮し、他のスタッフにも注意できる 基本的なルールは守る。年1〜2回注意を受ける程度 近隣からの苦情がある。騒音・マナー違反を繰り返し注意される
ルール遵守 ゴミ出し・分別・当番など寮のルールを率先して守る。他のスタッフにも声かけできる 言われれば守る。基本的なルールは理解している ルールを守らないことが多い。繰り返し注意が必要
住環境維持 部屋・共用部をいつもきれいに保つ。整理整頓が行き届いている 普通の清潔さは保っている。時々散らかっていることがある 部屋や共用部が不衛生。清掃を怠ることが多い

12. スコア計算と昇給テーブル

評価者の構成

カテゴリ評価者
日本語能力・勤務態度・職業能力職長3名 + 政仁さん + 靖仁さん
生活態度靖仁さんのみ（エムテック担当者と確認の上、代理入力）

スコア計算

各項目: A=3点、B=2点、C=1点

重み付きスコア:
日本語能力（3項目の合計）× 1.0 = 最大 9.0点（20%）
+ 勤務態度（4項目の合計）× 1.5 = 最大 18.0点（40%）
+ 職業能力（3項目の合計）× 1.0 = 最大 9.0点（20%）
+ 生活態度（3項目の合計）× 1.0 = 最大 9.0点（20%）
= 手動スコア（最大 45.0点）

出勤率ボーナス: 98%以上→+3点 / 95%以上→+2点 / 90%以上→+1点

総合スコア = 手動スコア + 出勤率ボーナス（最大 48.0点）
勤務態度の配点が最大（40%）を占めます。
技能がまだ発展途上の実習生でも、勤務態度がしっかりしていれば高評価（A〜S）を取ることができます。「真面目にやる人が報われる」仕組みです。

13. よくある質問・トラブル対処

質問・症状回答・対処
提出ボタンを押したが反映されたか分からない2026-05-09 改善で、提出成功時には 確認ダイアログ＋緑トースト＋「✓ 提出済み」バナー（提出時刻付き）が表示されます。これらが出ていなければ保存されていません。失敗時は ❌ 提出に失敗しました のアラートが出るので、もう一度押してください。
評価入力タブにスタッフが表示されないあなたが評価者に指定されていない可能性があります。管理者に確認してください。
他の評価者の結果が見えない自分が提出するまで、他の人の評価は非表示です（バイアス防止のため）。先に自分の評価を提出してください。
提出した後に修正したい「修正する」ボタンで再編集・再提出できます。
承認タブに何も表示されない全員の提出が完了するまで承認画面には表示されません。「一覧」タブの提出状況バッジ、または「進捗監視」タブで状況を確認してください。
承認済みの評価をもう一度見たい「履歴」タブで承認済み評価を一覧表示できます。一覧タブのランク文字をクリックしても詳細モーダルが開きます。
大川さんなど特定の職長の評価だけ反映されない「進捗監視」タブで該当セッションの評価者バッジを確認してください。橙色（7日以上未提出）になっていればリマインダーが既に届いているはず。本人に再提出を依頼してください。
ウェイトを最新の出面データで更新したい一覧タブ右上の 🔄 ウェイト一括再計算 ボタン、または詳細モーダル内の 🔄 再計算 ボタンで実行できます。承認済みは対象外です。
ウェイト 0.30 になっている職長が居るその職長は対象スタッフと過去365日に共働実績がありません（最低ウェイト）。承認時の最終ABC決定で、その意見を参考程度に扱う運用がおすすめです。
昇給額はどう反映される？評価承認後、管理者が人員マスタの時給を手動で更新します。
評価の頻度は？年1回、各スタッフの入社日基準です。入社日の1ヶ月前に通知が届きます。
旧データとの互換性は？「指示遵守」項目がない旧評価データはB（標準）として計算されます。ウェイト未計算の旧セッションは全員 weight=1.0 で扱われます（一括再計算で更新可能）。
